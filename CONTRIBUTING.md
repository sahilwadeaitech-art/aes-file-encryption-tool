# Contributing

If you want to contribute, here's the basics:

1. Fork and clone
2. `pip install -r requirements.txt`
3. Make your changes on a feature branch
4. Test it — at minimum run `python -m pytest tests/`
5. Open a PR with a clear description

## Style

- PEP 8 mostly, nothing crazy
- Type hints where it makes sense
- Keep commits focused (one thing per commit)

## Good contributions

- Bug fixes
- Cross-platform fixes (especially Linux/macOS font issues)
- UI improvements
- Performance improvements for large files
- Tests

## Please don't

- Add offensive security / exploit features
- Add heavy dependencies without discussion
- Make breaking changes without opening an issue first

Questions? Open an issue.
